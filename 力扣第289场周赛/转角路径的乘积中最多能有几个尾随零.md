### 转角路径的乘积中最多能有几个尾随零

[https://leetcode-cn.com/problems/maximum-trailing-zeros-in-a-cornered-path/](https://leetcode-cn.com/problems/maximum-trailing-zeros-in-a-cornered-path/)

### 题目描述

<font size=2> 

给你一个二维整数数组 grid ，大小为 m x n，其中每个单元格都含一个正整数

**转角路径** 定义为：包含至多一个弯的一组相邻单元。具体而言，路径应该完全 **向水平方向** 或者 **向竖直方向** 移动过弯（如果存在弯），而不能访问之前访问过的单元格。在过弯之后，路径应当完全朝 **另一个** 方向行进：如果之前是向水平方向，那么就应该变为向竖直方向；反之亦然。当然，同样不能访问之前已经访问过的单元格

一条路径的 **乘积** 定义为：路径上所有值的乘积

请你从 grid 中找出一条乘积中尾随零数目最多的转角路径，并返回该路径中尾随零的数目

注意：

</font>

- <font size=2>**水平** 移动是指向左或右移动</font>
- <font size=2>**竖直** 移动是指向上或下移动</font>

### 题解

#### 方法一

- <font size=2>尾数零的个数就是路径上的数字所含质因子 2 的个数和质因子 5 的个数中的较小值，即min{num{2}，num{5}}。比如 15, 12, 4, 3, 7五个数中，包含了 1 个质因子 5，4 个质因子 2，所以应该有 1 个尾数零（15120）</font>
- <font size=2>那么路径上的数字自然是越多越好，所以路径的起点和终点都应该在边界上</font>
- <font size=2>接下来可以预处理 **每一行** 质因子 2 和质因子 5 的前缀和，然后枚举每一个点作为拐角，如下图有四种情况</font>

- <font size=2> 由上图可知，我们需要从上往下，枚举左拐/右拐（1、2图），接着需要从下往上，枚举左拐/右拐（3、4图）。代码实现上我们是先枚举 **每一列**，再枚举每一行。枚举行的时候进行竖直方向的累加，根据前缀和我们就可以求出当前路径下的尾随零是多少，从而取 max 即可求得答案</font>

#### AC代码

```
class Solution {
    static int[][] c25 = new int[1001][2];

    static {
        // 预处理：递推算出每个数的因子 2 的个数和因子 5 的个数
        for (var i = 2; i <= 1000; i++) {
            if (i % 2 == 0) c25[i][0] = c25[i / 2][0] + 1;
            if (i % 5 == 0) c25[i][1] = c25[i / 5][1] + 1;
        }
    }

    public int maxTrailingZeros(int[][] grid) {
        int m = grid.length, n = grid[0].length, ans = 0;
        var s = new int[m][n + 1][2];
        for (var i = 0; i < m; i ++)
            for (var j = 0; j < n; j ++) {
                s[i][j + 1][0] = s[i][j][0] + c25[grid[i][j]][0]; // 每行的因子 2 的前缀和
                s[i][j + 1][1] = s[i][j][1] + c25[grid[i][j]][1]; // 每行的因子 5 的前缀和
            }

        for (var j = 0; j < n; j ++) {
            for (int i = 0, s2 = 0, s5 = 0; i < m; i ++) { // 从上往下，枚举左拐还是右拐
                s2 += c25[grid[i][j]][0];
                s5 += c25[grid[i][j]][1];
                ans = Math.max(ans, Math.max(Math.min(s2 + s[i][j][0], s5 + s[i][j][1]), 
                                             Math.min(s2 + s[i][n][0] - s[i][j + 1][0], s5 + s[i][n][1] - s[i][j + 1][1])));
            }

            for (int i = m - 1, s2 = 0, s5 = 0; i >= 0; i --) { // 从下往上，枚举左拐还是右拐
                s2 += c25[grid[i][j]][0];
                s5 += c25[grid[i][j]][1];
                ans = Math.max(ans, Math.max(Math.min(s2 + s[i][j][0], s5 + s[i][j][1]), 
                                             Math.min(s2 + s[i][n][0] - s[i][j + 1][0], s5 + s[i][n][1] - s[i][j + 1][1])));
            }
        }

        return ans;
    }
}
```

#### 复杂度分析

- <font size=2>时间复杂度：O(n^2)</font>
- <font size=2>空间复杂度：O(n^3)</font>
