### 完成所有任务需要的最少轮数

[https://leetcode-cn.com/problems/minimum-rounds-to-complete-all-tasks/](https://leetcode-cn.com/problems/minimum-rounds-to-complete-all-tasks/)

### 题目描述

<font size=2> 

给你一个下标从 **0** 开始的整数数组 tasks ，其中 tasks[i] 表示任务的难度级别。在每一轮中，你可以完成 2 个或者 3 个 **相同难度级别** 的任务

返回完成所有任务需要的 **最少** 轮数，如果无法完成所有任务，返回 -1 

</font>

<font size=2>**示例 1：**</font>

```
输入：tasks = [2,2,3,3,2,4,4,4,4,4]
输出：4
解释：要想完成所有任务，一个可能的计划是：
- 第一轮，完成难度级别为 2 的 3 个任务。 
- 第二轮，完成难度级别为 3 的 2 个任务。 
- 第三轮，完成难度级别为 4 的 3 个任务。 
- 第四轮，完成难度级别为 4 的 2 个任务。 
可以证明，无法在少于 4 轮的情况下完成所有任务，所以答案为 4 。
```

<font size=2>**示例 2：**</font>

```
输入：tasks = [2,3,3]
输出：-1
解释：难度级别为 2 的任务只有 1 个，但每一轮执行中，只能选择完成 2 个或者 3 个相同难度级别的任务。因此，无法完成所有任务，答案为 -1 。
```

<font size=2>**提示：**</font>

- `1 <= tasks.length <= 10^5`
- `1 <= tasks[i] <= 10^9`

### 题解

#### 方法一、分类讨论 + 哈希表

- <font size=2>用哈希表记录每种任务的数量</font>
- <font size=2>接下来考虑每种任务，按照任务的数量分类讨论：</font>
  - <font size=2>如果该任务只有 1 个，那么无法完成</font>
  - <font size=2>如果该任务只有 2 个，1 次完成即可</font>
  - <font size=2>如果该任务数是 3 的倍数，那么显然每次完成 3 个是最快的</font>
  - <font size=2>如果该任务数除 3 余 1，需要每次完成 3 个，最后 4 个分 2 次完成</font>
  - <font size=2>如果该任务数除 3 余 2，需要每次完成 3 个，最后 2 个 1 次完成</font>

- <font size=2>可以发现，有些情况的代码可以合并，具体可见下面</font>

#### AC代码

```
class Solution {
    public int minimumRounds(int[] tasks) {
        HashMap<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < tasks.length; i ++) {
            map.put(tasks[i], map.getOrDefault(tasks[i], 0) + 1);
        }

        int res = 0;
        for (int num : map.values()) {
            if (num == 1) return -1;
            else if (num % 3 == 0) res += num / 3;
            else res += num / 3 + 1;
        }

        return res;
    }
}
```

#### 复杂度分析

- <font size=2>时间复杂度：O(n)</font>
- <font size=2>空间复杂度：O(n)</font>
